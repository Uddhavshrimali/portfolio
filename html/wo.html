Here is a list of common Git commands and a brief description of what they do: 1. `git init`: Initializes a new Git repository in the current directory. 2. `git clone `: Creates a copy of a remote Git repository on your local machine. 3. `git add `: Stages changes to a file for the next commit. 4. `git commit -m ""`: Commits the staged changes to the repository with a descriptive message. 5. `git status`: Shows the status of your working directory and staged changes. 6. `git log`: Displays a log of all commits in the repository. 7. `git branch`: Lists all branches in the repository, with an asterisk indicating the current branch. 8. `git checkout `: Switches to a different branch. 9. `git merge `: Merges changes from one branch into the current branch. 10. `git pull`: Fetches changes from a remote repository and merges them into the current branch. 11. `git push`: Pushes your local commits to a remote repository. 12. `git remote -v`: Lists the configured remote repositories. 13. `git diff`: Shows the differences between your working directory and the last commit. 14. `git stash`: Temporarily saves changes that are not ready for commit. 15. `git reset `: Unstages changes from the index for a specific file. 16. `git remote add `: Adds a new remote repository with a given name and URL. 17. `git remote remove `: Removes a remote repository. 18. `git tag`: Lists all tags in the repository. 19. `git tag -a -m ""`: Creates an annotated tag. 20. `git branch -d `: Deletes a branch. 21. `git fetch`: Retrieves changes from a remote repository without merging. 22. `git push origin --delete `: Deletes a remote branch. 23. `git rebase `: Reapplies your local changes on top of another branch. 24. `git config --global user.name "Your Name"`: Sets your global Git username. 25. `git config --global user.email "youremail@example.com"`: Sets your global Git email address. These are some of the most commonly used Git commands. Git offers a wide range of functionalities for version control and collaboration, so you may need to use additional commands for more specific tasks. You can always get more information about a command by running `git --help` or `git --help`.
